Important

Finding Your Config File

Before anything else — know where your config file lives. The guide references ~/.openclaw/openclaw.json everywhere. Here's what that actually means on your computer:

Windows
C:\Users\YOUR_USERNAME\.openclaw\openclaw.json
Mac
/Users/YOUR_USERNAME/.openclaw/openclaw.json
Linux
/home/YOUR_USERNAME/.openclaw/openclaw.json
Windows shortcut: Press Win + R, type %USERPROFILE%\.openclaw, press Enter. That opens the folder. Look for openclaw.json. If it doesn't exist, run openclaw configure in your terminal to create it.

The file is JSON5 format — it supports comments (//) and trailing commas, so it's forgiving to edit. There is only ONE config file: openclaw.json. Not config.json, not config.yaml, not .env — just openclaw.json.

Your workspace files (AGENTS.md, SOUL.md, etc.) go in a different folder:

Windows Workspace
C:\Users\YOUR_USERNAME\.openclaw\workspace\
Mac/Linux Workspace
~/.openclaw/workspace/
Getting Started

Quick Start — 15 Minutes

1

Install OpenClaw

Run npm install -g openclaw then openclaw configure and follow the wizard. It connects your AI model provider (Anthropic, OpenAI, etc.) and creates your config file.

2

Copy the Workspace Files

Copy everything from the workspace/ folder into your workspace directory. See the paths above — on Windows it's C:\Users\YOU\.openclaw\workspace\

3

Fill In the Brackets

Open each file and replace the [BRACKETS] with your info. Start with USER.md (about you), then SOUL.md (agent personality), then IDENTITY.md (agent name).

4

Configure the Memory System (Critical!)

Open your openclaw.json config file and add the memory settings. Use the example config included in this kit (docs/example-config.json5) as a starting point. The three critical settings are: memoryFlush.enabled: true, postCompactionSections, and heartbeat.every.

5

Start Chatting

Run openclaw gateway start and open the dashboard at http://127.0.0.1:18789. Your agent reads your workspace files automatically and knows who it is, who you are, and how to behave.

Tip: Don't try to perfect everything on day one. Set up the basics, start talking to your agent, and refine over time. It learns your preferences naturally through conversation.

Compaction: What Happens With Long Conversations

Every AI model has a "context window" — a limit on how much text it can hold. When your conversation fills up, OpenClaw compacts it: older messages get summarized to free space.

This is where most AI setups break. Without proper configuration, compaction = amnesia. Here's how we prevent that:

💾

Memory Flush

Before compaction, your agent is asked to save anything important to the daily log. This is the safety net — notes survive even when the conversation doesn't.

🔄

Post-Compaction Sections

After compaction, key sections from AGENTS.md are re-injected so your agent knows: how to start up, how memory works, and safety rules. Without this, it wakes up confused.

🔍

Memory Search

Your agent can search across all daily logs and memory files — even months old. Ask "what did we decide about X?" and it finds the answer instantly.

The three settings that make this work:
1. memoryFlush.enabled: true — saves notes before compaction
2. postCompactionSections: ["Every Session", "Memory", "Safety"] — re-injects instructions after
3. heartbeat.every: "30m" — periodic check-ins to catch anything missed

Without all three, your agent will lose context over time. The example config in this kit has them all set up.
Configuration

The Config File

This is the most important part. Your config file controls whether the memory system actually works.

An example config (docs/example-config.json5) is included in this kit. Here are the essential settings:

// In your openclaw.json, under agents.defaults:
{
  agents: {
    defaults: {
      compaction: {
        mode: "safeguard",
        memoryFlush: {
          enabled: true,
          prompt: "Write any lasting notes to memory/YYYY-MM-DD.md",
          systemPrompt: "Session nearing compaction. Store durable memories now."
        },
        // CRITICAL: These sections from AGENTS.md get re-injected
        // after compaction. Must match your ## headers exactly.
        postCompactionSections: ["Every Session", "Memory", "Safety"]
      },
      heartbeat: {
        every: "30m"
      },
      contextPruning: {
        mode: "cache-ttl",
        ttl: "1h"
      }
    }
  }
}
SettingWhat It DoesRecommended
memoryFlush.enabledSaves memories before context resetstrue (required!)
postCompactionSectionsRe-injects instructions after compaction["Every Session", "Memory", "Safety"]
heartbeat.everyHow often the agent checks in"30m" to "1h"
contextPruning.modeHow old context is handled"cache-ttl"
memorySearch.providerHow memory is searched"gemini" (free!) or "local"
postCompactionSections explained: After compaction wipes your agent's short-term memory, these named sections from AGENTS.md get automatically re-injected. The names must match the ## Section Header names in your AGENTS.md exactly (case-sensitive). In the included AGENTS.md, the key headers are "Every Session", "Memory", and "Safety". If you rename them, update this config to match.

Advanced

Lossless Context (LCM Plugin)

Optional but powerful. LCM makes compaction lossless — instead of permanently losing old messages, they're stored as searchable summaries that can be expanded back to full detail.

Without LCM

When compaction happens, older messages are summarized and the originals are gone. Your agent's manual notes (daily logs) are the only safety net.

With LCM

Compacted messages are stored in a local database. Your agent can search across all past conversations and expand summaries back to the original detail. Nothing is permanently lost.

How to Install

openclaw plugin install @martian-engineering/lossless-claw

Then add to your config:

plugins: {
  entries: {
    "lossless-claw": {
      enabled: true
    }
  }
}
Is LCM required? No. The basic memory system (MEMORY.md + daily logs + memory flush) works great without it. LCM is a nice-to-have that prevents information loss during compaction. Start without it and add it later if you want. The memory flush + postCompactionSections combo handles 90% of cases.

Help

Quick Troubleshooting

Agent doesn't remember?

Check memoryFlush.enabled: true AND postCompactionSections in config. Both are needed.

Can't find config file?

Windows: Win+R → %USERPROFILE%\.openclaw → look for openclaw.json. Run openclaw configure if missing.

Agent confused after long chats?

Compaction wiped context. Check that postCompactionSections matches your AGENTS.md headers exactly.

Wrong personality?

Edit SOUL.md, be more specific, start a new session. Old sessions cache the old personality.

No daily logs?

Create the memory/ directory in your workspace. The agent can't create it if it doesn't exist.

Need more help?

Run openclaw doctor to check your setup, or visit the community.
